Responsive Redesign: Online Voting Information

Overview

The goal of this project was to work through all steps of redesigning a website, from identifying usability issues through developing a responsive website. I learned the importance of each step of the redesign process and used Bootstrap for the first time! This project was an exciting opportunity to apply what I have learned in class to a website in my local community.

Chosen Website

The website I chose is the elections page of my hometown's government website. With the upcoming election and the COVID-19 pandemic, I thought it would be especially important that voting information is easy to find online. I remembered my difficulties finding information about how to vote early in my hometown and decided that this would be the perfect website to redesign.

Finding Problems

Usability

After choosing my website, I analyzed its usability and found a few problems which I have outlined below.

  • Usability - a nearly effortless understanding of the architecture and navigation of the site
    • Not intuitive that the top four buttons will link to external website
    • Inconsistent information hierarchy and header style
    • Poor information architecture and unclear what is emphasized
  • Learnability - how fast a user who has never seen the user interface before can accomplish basic tasks
    • Repeated information on different parts of the website makes it hard to know where to go
  • Memorability - after visiting the site, if a user can remember enough to use it effectively in future visits
    • Other than the four main buttons, there is an overwhelming amount of text which makes it hard to remember exactly what is on the page and where it is
    • Many side navigation buttons makes it hard to remember how to get to information again
  • Efficiency - How fast an experienced user can accomplish tasks
    • Long scroll makes it hard to find desired information
    • Even someone experienced would most likely have to use a command F function or scroll for a while

Accessibility

Next I analyzed the website from an accessibility framework. I used the WebAIM Wave tool and the built in Mac screenreader tool (Voiceover). The accessibility problems I found are laid out below

  • Contrast issues - light green text color on white background
  • Missing elements - lacking alternative text for images, empty headers, and empty links
  • Poor structure/hierarchy - using headers is not just for visual appeal, but also an accessibility issue
    • Because this page has almost no headers used in the text body of the page it was very difficult to navigate using the screen reader which relies on cues from the html

Lo-fi Wireframes

I started the redesign process by creating lo-fi wireframes in Balsamiq that addressed the key usability issues outlined above. My solution was driven by three main principles.

  • Simplicity
    • Getting rid of the side nav bar
    • Only include important voting information
    • Get rid of repeated information
  • Efficiency
    • Condense long scroll text into buttons that will link out to subpages
    • Make the page easier to navigate and accomplish tasks quickly
  • Consistency
    • Develop consistent headers, subheadings, and information architecture

These changes are annotated on my lo-fi wireframes below. I created a wireframe for each device size to demonstrate responsiveness.

Hi-fi Prototypes

I built on my lo-fi wireframes by creating hi-fi prototype in Adobe XD. Again I made a prototype for each device to demonstrate responsiveness.

I annotated the prototypes with important html elements to show how I planned to build the responsive functionality. I used bootstrap to develop so some functionality is built in, but I tried to highlight how the site changes across each device.
